System Requirements


Select HERE for Help on Course Navigation and Functions

The IMI courseware is designed to run on an IBM-PC, Macintosh desktop, or mobile device (Apple or Android) as described below with a 2.0 GHz CPU or higher clock speed with the following minimum requirements:

  • 1 Gb or greater of RAM
  • Graphics card is either 32 bit PCI or AGP
  • Hard drive with at least 1 Gbyte of free storage area
  • Direct X 9.0 compatible or integrated on board (external speakers recommended)
  • The following browsers − IE version 11.0, MS Edge and Safari 5.x, 6.x, 7.x for desktop devices and for mobile devices Safari 5.0, 5.1, 8536, 6.0, 7.0 and Android 5.0, 4.0.
  • The following desktop and laptop operating systems: Windows 7 or newer and OS 10.6, 10.7, 10.8, 10.9 including the latest service packs and security patches available
  • The following mobile device operating systems: iOS 3.1.3, 4.2.1, 5.1.1, 6.x, 7.5 and Android 2.3, 3.0, 4.0-3.
  • Broadband access of 512kb/s
  • Multimedia-computer with CD-ROM drive (quadruple speed (4X) or faster with maximum access time of 250ms
  • Appropriate language pack for use with language keyboard entry

Display Color Depth

This course looks best with a color depth setting higher than 256 colors (8 bit). To change the color depth of the display, follow the above steps for changing the display monitor resolution. When on the Settings tab, look for the Color Palette, Color Depth, or Color Quality area. Change the color setting to High Color (16 bit), True Color (24 bit), or True Color (32 bit) and select OK.

    Printing


    To print the activity you are currently working on, click on the Tool Bar Print Button. Another option is to select Ctrl + P. A dialogue box will appear. Choose the printer you want to print from and select Print. The computer will print an approximation of the screen you are viewing. If you would like to include typically off-screen information, such E-Mentor notes, select the note in question to make it visible before you select Print.

    Troubleshooting


    Install Setup Errors

    If your computer does not have the correct components installed, the Course setup will automatically guide you through installation of needed software. However, if you are on a restricted Government computer and receive error messages or experience problems with your installation, you most likely lack privileges necessary to install software on your computer. Please contact your Administrator and have him or her install the program for you.


    Audio

    If you are having problems hearing audio, first try opening Windows ® Volume Controls. Many systems can access these controls via a speaker icon on the Windows ® taskbar next to the clock. It may also be found by opening the Start Menu, then selecting Programs, Accessories, Entertainment, and Volume Control. Once you have opened the volume controls, make sure that the Volume Control and Wave controls are not muted.

    If these controls are not shown, try selecting Options, then Properties from the Volume Control menu bar. At the bottom of this new window, make sure that Master Volume and Wave boxes are all checked, then select OK.

    Another possibility is that the audio drivers are not properly set up; check these in the Sounds and Audio Devices (in Windows ® XP or newer) section of your Control Panel. If your speakers/headphones are turned on and properly connected and these steps do not resolve the problem, contact your System Administrator.

    If you are experiencing problems in the playback of audio segments – for example, the audio does not play correctly, is clipped at the beginning, or is skipping portions of audio segments, your hardware acceleration settings may need to be reduced.

    To correct this in Windows ® XP or newer:

    • Go to Control Panel
    • Select Sounds, Speech, and Audio Devices
    • Select Change Sound Scheme
    • Select the Audio tab
    • Under Sound Playback, select Advanced
    • Select the Performance tab
    • Move the Hardware Acceleration slider closer to None

    Check if the problem has been solved. If not, continue moving the Acceleration slider closer to None until the problem is resolved.
